

# Mail Servers

You might find a full-featured internal mail server running on Windows, supporting SMTP, POP3, and IMAP protocols, with user mailboxes accessible over plaintext and SSL ports. Like Offsec PG `Heptet`

Typical Setups You Might See Together ( combined ports)
- `143 + 993 + `587 → **More modern** and secure mail stack
- `143 + 993` → **IMAP** (plain + secure)
- `25 + 465 + 587` → **SMTP** variants (send-only, secured)
- `110 + 995` → **POP3** (plain + secure)
- `143 + 25 + 110` → All plaintext, internal or older systems

# 📬 Mail Server Ports 
## (25, 79, 101,105,106,110,143,465,587,993,995)

```
└──📤 Sending Mail (SMTP)
   ├── 25    - SMTP (basic, relaying)
   ├── 465   - SMTPS (SMTP over SSL)
   └── 587   - SMTP (with STARTTLS, modern secure submission)

└──📥 Reading Mail
    ├── IMAP (Interactive, messages stay on server)
    │   ├── 143   - IMAP (plaintext)
    │   └── 993   - IMAPS (IMAP over SSL)
    │
    └── POP3 (Downloads and deletes mail)
        ├── 110   - POP3 (plaintext)
        └── 995   - POP3S (POP3 over SSL)

└──🧰 Utility / Legacy Services
    ├── 106   - poppassd (change POP3 password)
    ├── 79    - finger (user enumeration)
    └── 101/105 - older or vendor-specific mail mgmt
```

# ⚠️ IMAP 143 and IMAPS 993 Flow - Interrogation
Port 143 is the standard, unencrypted port for the Internet Message Access Protocol (IMAP), which email clients use to retrieve and manage emails from a remote server. It allows for synchronized access across multiple devices, unlike POP3 which typically downloads emails to a single device. For secure, encrypted connections, IMAP uses port 993 (IMAPS). 

## 0. Connect to the IMAP port
```
telnet 192.168.102.227 143
```
- response like: `* OK localhost IMAP4rev1 Mercury/32 v4.62 server ready.`

## 1. Login
```
a1 LOGIN USERNAME PASSWORD
```
- Response like : `a1 OK LOGIN completed.`
- `a1` is an identifying tag (because of the asynchronous nature of imap, it allows tracking of commands) . `bokko1`, `1abc`, `ab2c` etc could also be used.

## 2. List all the Mail boxes
```telnet
a2 LIST "" "*"
```

**Output:**
```
* LIST (\NoInferiors) "/" INBOX
a2 OK LIST completed.
```

- `* LIST` — server's response to your LIST command
- `(\NoInferiors)` — means this folder does not contain subfolders
- `"/"` — the hierarchy delimiter, used if subfolders exist (like INBOX/Archive)
- `INBOX` — the **name of the folder**

## 3. Select the inbox 
```
a3 SELECT INBOX
```

## 4. Read the mail - examples
### a. Select all emails subject lines
```
a4 FETCH 1:* (BODY[HEADER.FIELDS (SUBJECT)])
```
### b. Select email 1's body
```
a6 FETCH 1 BODY[]
```
### c. Select email subject and body for emails 1 - 4
```
a4 FETCH 1:4 (BODY[HEADER.FIELDS (SUBJECT)] BODY[TEXT])
```

# ⚠️ 📬 SMTP (Simple Mail Transfer Protocol)

### ✅ Common Ports
- `25` — Plain SMTP (no encryption, default for server-to-server)
- `587` — SMTP with STARTTLS (modern secure submission)
- `465` — SMTP over SSL (legacy, still used)

---

### 🔍 Purpose
- SMTP is used to **send** email between clients and servers, or from server to server.
- It's **not** for reading messages (that's IMAP/POP3).
- Ideal for:
    - **User enumeration**
    - **Testing open relays**
    - **Spoofing / injection attacks**
    - **Sending crafted emails (phishing, payload delivery)**

Try for usernames
```
smtp-user-enum -M VRFY -U /usr/share/seclists/Usernames/Names/names.txt -t 192.168.102.227
```

---

## 🔧 Manual Enumeration via Telnet

```
telnet 192.168.102.227 25
```
**Example flow:**
```
EHLO attacker.com
MAIL FROM:<spoof@attacker.com>
RCPT TO:<target_user@localhost>
DATA
Subject: Test

This is a test message.
.
QUIT
```
---

### 🧪 SMTP Commands for Recon
| Command         | Purpose                                   |
| --------------- | ----------------------------------------- |
| `HELO` / `EHLO` | Greet the server, see capabilities        |
| `VRFY <user>`   | Verify if a user/mailbox exists           |
| `EXPN <list>`   | Expand a mailing list (if allowed)        |
| `RCPT TO:`      | Can be used to test if recipient is valid |
| `MAIL FROM:`    | Define sender (can be forged)             |
| `DATA`          | Begin message input                       |
| `AUTH LOGIN`    | Try to auth (often base64 encoded)        |
| `QUIT`          | Close connection                          |

---

### 📍 Notes on Behavior
- `VRFY` and `EXPN` may be **disabled** or return generic responses
- `RCPT TO:` may still **leak valid usernames**
- Servers may require a **valid `MAIL FROM`** before accepting `RCPT TO`
- Use full addresses: `<user@localhost>` or `<user@example.com>`

---

## 🧰 Enumeration Tools

| Tool                               | Use                                                          |
| ---------------------------------- | ------------------------------------------------------------ |
| `smtp-user-enum`                   | Automates `VRFY`, `RCPT TO` attacks                          |
| `swaks`                            | Swiss Army Knife for SMTP — great for scripted email sending |
| `telnet` / `nc`                    | Manual testing                                               |
| `nmap -p25 --script smtp-commands` | Pulls EHLO info and checks for auth support                  |

## 💣 Exploitation Tactics

### ✅ Injecting Payloads
Send a phishing payload or macro doc:
```
sendemail -f attacker@evil.com -t victim@target.com -s 192.168.102.227:25 \ -u "Your document" -m "See attached" -a payload.ods
```
Or use `swaks`:
```
swaks --to victim@target.com --from you@spoof.com \ --server 192.168.102.227 --attach payload.ods --data "Here is the doc"
```


---

### 🔍 After Getting Shell — Look for Local Mail:
```
cat /var/mail/* 
cat /var/spool/mail/*
```
May contain:
- Password reset links
- Internal system alerts
- Developer notes
- Auto-forwarded credentials

---

### 🧠 Testing for Open Relay
```
telnet 192.168.102.227 25 HELO test.com MAIL FROM:<you@external.com> RCPT TO:<other@external.com>
```

If it accepts this **without auth**, it's an **open relay** — dangerous misconfig.

# ⚠️ 📥 POP3 (Post Office Protocol v3)

### ✅ Common Ports
- `110` — Default POP3 (plaintext)
- `995` — POP3S (POP3 over SSL)

---

### 🔍 Purpose
- POP3 is used by email clients to **retrieve and delete messages** from a remote mail server.
- Unlike IMAP, POP3 usually **downloads** emails and **removes** them from the server afterward.
- It’s a **sequential protocol** — one command at a time, no tags.

---

## 🧪 Manual Interrogation (Telnet)

```
telnet 192.168.102.227 110
```

### 🧾 Basic POP3 Session

```
+OK POP3 server ready
USER jonas
+OK
PASS SicMundusCreatusEst
+OK User successfully logged on

LIST
+OK 5 messages
1 1354
2 1420
...

RETR 1
+OK 1354 octets
Subject: Important
From: admin@localhost

This is the full email body.
.

QUIT

```

---

### 🧠 POP3 Command Cheatsheet

|Command|Description|
|---|---|
|`USER`|Start login with username|
|`PASS`|Send password|
|`CAPA`|Get server capabilities|
|`LIST`|List message IDs and sizes|
|`RETR <n>`|Retrieve full message number `<n>`|
|`DELE <n>`|Mark message `<n>` for deletion|
|`NOOP`|Do nothing (keep alive)|
|`STAT`|Show number of messages + size|
|`RSET`|Unmark all messages for deletion|
|`TOP <n> <lines>`|Get headers + first `<lines>` of body|
|`QUIT`|End session|

---

### 🔍 Output Examples

#### `LIST`
```
+OK 3 messages: 1 1024 2 1876 3 1248
```

#### `RETR 1`
```
+OK From: mailadmin@localhost To: jonas@localhost Subject: Update  Here’s the new login info. .
```

#### `TOP 1 5` (Preview only)
```
+OK From: admin@localhost To: jonas@localhost Subject: Important Date: ...  <First 5 lines of the body> .
```


## 🧰 Useful Tools
| Tool                                 | Purpose                       |
| ------------------------------------ | ----------------------------- |
| `telnet` / `nc`                      | Manual inspection             |
| `openssl s_client -connect 192.168.102.227:995` | POP3S manual access           |
| `getmail` / `fetchmail`              | Download mail to local system |
| `pop3-brute`, `hydra`                | Brute-force passwords         |
| `nmap --script pop3-capabilities`    | See server features           |

### 💣 Attack Vectors
- **Brute force**: Try weak credentials (`hydra`, `medusa`, `nmap`)
- **Credential reuse**: If you have a password from web or SMB
- **Plaintext leaks**: Users sometimes receive secrets in mail
- **Phishing targets**: Use legit sender in email headers for trust

## 📦 After Shell — Check for Local Mail

If the server downloads POP3 mail to local files:

```
cat /var/mail/* cat /var/spool/mail/*
```

Look for:
- Full plaintext messages
- Credentials or internal alerts
- Sensitive conversations

## 🛑 Limitations of POP3

|Limitation|IMAP Alternative|
|---|---|
|One folder: INBOX only|IMAP supports multiple|
|Deletes messages on retrieval|IMAP can sync & archive|
|No server-side search|IMAP supports SEARCH|
|No tagging or flagging|IMAP supports flags|
